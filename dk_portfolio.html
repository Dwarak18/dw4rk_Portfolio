01 — About

The discipline
behind the craft.

● Curiosity
Started taking systems apart before understanding how they worked. CTF competitions, local exploits, late-night rabbit holes into kernel internals and protocol specs.
● Confusion
Realized that knowing how to break things doesn't automatically mean knowing how to build them right. Started bridging the gap — architecture, system design, product thinking.

The intersection became clear: use offensive knowledge to build defensively strong systems. Bug bounty programs validated the model. Startups became the output.
● Now
Building Bl4ckdot. Running active bounty campaigns. Shipping tools. The goal hasn't changed — understand systems completely, then make them unbreakable.
